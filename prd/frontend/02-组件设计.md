## 二、通用组件详细设计

### 2.1 ChatPanel.vue — 对话面板（🔧 改造）

```
┌─────────────────────────────────────────────────┐
│  ChatPanel                                       │
│  ┌─────────────────────────────────────────────┐│
│  │ 消息列表 (v-for)                              ││
│  │  ┌─────────────────────────────────────┐    ││
│  │  │ 👤 用户: "什么是层次分析法？"          │    ││
│  │  └─────────────────────────────────────┘    ││
│  │  ┌─────────────────────────────────────┐    ││
│  │  │ 🤖 Agent: "层次分析法(AHP)是..."      │    ││
│  │  │  ├─ Markdown渲染 (markdown-it)       │    ││
│  │  │  ├─ LaTeX公式 (KaTeX)                │    ││
│  │  │  ├─ 代码块 (CodeBlock)               │    ││
│  │  │  └─ 追问建议 (chip标签)              │    ││
│  │  └─────────────────────────────────────┘    ││
│  └─────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────┐│
│  │ 输入区                                        ││
│  │ ┌──────────────────────────┐ ┌────────────┐ ││
│  │ │ 输入消息...               │ │ 发送 →     │ ││
│  │ └──────────────────────────┘ └────────────┘ ││
│  └─────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘

Props:
  - agentType: string        // 当前Agent类型
  - sessionId: string        // 会话ID
  - messages: Message[]      // 消息列表 {role, content, time}

Events:
  - @send(message: string)   // 发送消息
  - @stop()                  // 停止流式生成

🆕 改造点:
  1. 支持流式渲染（逐token追加到最新消息）
  2. 追问建议以chip标签显示在每条回复末尾，点击即发送
  3. Markdown代码块内嵌 CodeBlock 组件（复制/运行按钮）
  4. 加载状态：LoadingSkeleton 骨架屏
```

### 2.2 CodeBlock.vue — 代码块组件 🆕

```
Props:
  - code: string             // 代码内容
  - language: string         // 语言 (默认python)
  - runnable: boolean        // 是否可运行 (默认true)

Slots: —

Features:
  - 语法高亮 (highlight.js)
  - 一键复制 (navigator.clipboard)
  - "在编辑器中打开" 按钮 → 跳转 Workspace
  - "运行" 按钮 → 调用沙箱API，展示结果弹窗
```

### 2.3 StatusBadge.vue — 状态标签 🆕

```
Props:
  - status: string           // pending/running/completed/failed/skipped
  - size: 'small'|'default'

显示效果:
  pending   → ⬜ 待执行 (灰色)
  running   → 🟡 执行中 (黄色+旋转动画)
  completed → 🟢 已完成 (绿色)
  failed    → ❌ 失败 (红色)
  skipped   → ⏭ 已跳过 (灰色虚线)
```

### 2.4 LoadingSkeleton.vue — 加载骨架屏 🆕

```
Props:
  - lines: number           // 骨架行数
  - animated: boolean       // 是否动画

用于: AgentChat 等待回复、知识库加载、竞赛阶段执行中
```

### 2.5 FileUploader.vue — 文件上传 🆕

```
Props:
  - accept: string           // 允许类型 ".pdf,.doc,.docx,.csv,.xlsx"
  - maxSize: number          // 最大文件大小(MB)
  - multiple: boolean

Events:
  - @upload(files: File[])   // 文件选择完成
  - @error(message: string)

Features:
  - 拖拽上传区域
  - 文件类型/大小校验
  - 上传进度条
```

## 四、React 组件

### 4.1 InteractiveChart.jsx — 交互图表 (P1)

```
基于 ECharts 的交互式数据可视化组件

Props:
  - data: object            // ECharts option 数据
  - height: number
  - interactive: boolean    // 是否可交互(缩放/拖拽/数据点选择)

功能:
  - 折线图、柱状图、散点图、热力图、3D图
  - 缩放、拖拽、tooltip
  - 导出PNG/SVG
```

### 4.2 ModelingCanvas.jsx — 建模画布 (P2)

```
基于 React Flow 的拖拽式建模流程图

功能:
  - 拖拽模型节点到画布
  - 连接节点表示数据流/模型关系
  - 导出为图片
  - 导出为 JSON 模型结构描述

节点类型:
  - 数据源节点 (CSV/Excel/API)
  - 处理节点 (清洗/变换/聚合)
  - 模型节点 (优化/预测/评价...)
  - 输出节点 (图表/表格/结论)
```

### 4.3 CodeEditor.jsx — Monaco编辑器增强 (P1)

```
基于 Monaco Editor 的增强版代码编辑器

功能:
  - Python 语法高亮 + 智能提示
  - 数学建模常用库的代码片段 (scipy.optimize, numpy, pandas...)
  - 变量悬停提示
  - 内联错误提示
  - 一键运行(对接沙箱API)

注意: Vue3 页面 CodeEditor.vue 已有基础编辑能力，
      React版是增强版，P1开发。
     MVP阶段使用 Vue3 的 CodeEditor.vue (基于 @monaco-editor/loader)。
```
