## 三、Workflow 桥接层详细设计

> 这是后端最核心的新增模块——连接教学平台层与论文引擎层。

### 3.1 模块关系

```
routers/competition.py
         │
         ▼
┌──────────────────────────────────────────────────┐
│          pipeline_orchestrator.py                 │
│          流水线编排器（主入口）                      │
│  · run_stage(stage, project_id) → StageResult    │
│  · run_auto(project_id, until="S7")              │
│  · get_status(project_id) → PipelineState        │
└──────┬──────────────┬──────────────┬─────────────┘
       │              │              │
       ▼              ▼              ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│skill_executor│ │contract_reader│ │stage_tracker │
│              │ │              │ │              │
│·run_script() │ │·read_json()  │ │·get_state()  │

│·get_output() │ │·list_files() │ │·validate()   │
└──────┬───────┘ └──────┬───────┘ └──────┬───────┘
       │                │                │
       ▼                ▼                ▼
  subprocess      paper_output/      Redis + DB

```

### 3.2 Skill 执行器 (`skill_executor.py`)

```python
class SkillExecutor:
    """执行 MathModel Skill 脚本或 Claude Code Agent"""

    async def run_python_script(
        self,
        script_path: str,       # "analyze_problem.py"
        args: list[str],        # ["--problem", "problem.pdf"]
        cwd: str,               # 工作目录
        timeout: int = 300,
        env: dict = None
    ) -> ExecutionResult:
        """asyncio.subprocess 执行 Python 脚本"""

    async def run_claude_agent(
        self,
        skill_name: str,        # "paper-workflow-orchestrator"
        prompt: str,
        working_dir: str,
        timeout: int = 600
    ) -> ExecutionResult:
        """通过 Claude Code CLI 执行 Agent Skill"""

    async def run_with_stream(
        self, script_path, args, cwd
    ) -> AsyncGenerator[str, None]:
        """流式执行，通过 WebSocket 推送实时输出到前端"""
```

## 四、技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Web 框架 | FastAPI | 已有代码基础，异步+WebSocket原生支持 |
| ORM | SQLAlchemy 2.0 异步 | 已有代码基础 |
| 数据库 | SQLite(开发) → PostgreSQL(生产) | MVP零部署成本 |
| 缓存 | Redis 7 | Agent会话上下文 + 工作流状态 |
| 代码沙箱 | Docker SDK for Python | 原生集成，安全隔离 |
| LLM 调用 | 自研 llm_service 多厂商适配 | 已有，支持OpenAI/DeepSeek/Qwen/GLM |
| Skill 执行 | asyncio.subprocess | 无额外依赖，可流式读取 |
| 契约格式 | JSON（与 MathModel Skill 一致） | 引擎层已定义完整 Schema |

### 安全性

| 关注点 | 措施 |
|------|------|
| LLM API 密钥 | 数据库存储+浏览器密钥模式（密钥不出浏览器） |
| 代码沙箱 | Docker: 禁用网络, CPU 1核, 内存512MB, 超时30s |
| 文件上传 | 限制类型(PDF/Word/CSV), 大小<50MB |
| Agent 提示注入 | System Prompt与用户输入分离 |
