## 二、路由守卫

### 2.1 现有守卫逻辑（保持）

```js
router.beforeEach((to, from, next) => {
  // 白名单：Login, Register → 直接放行
  if (WHITE_LIST.includes(to.name)) { next(); return }

  // 非白名单：检查 token
  const token = localStorage.getItem('token')
  if (!token) { next({ name: 'Login', query: { redirect: to.fullPath } }); return }

  // 角色权限检查
  if (to.meta.role && !to.meta.role.includes(user.role)) {
    next({ name: 'Dashboard' }); return
  }

  next()
})
```

### 2.2 🆕 新增守卫规则

```js
// 竞赛模式专属守卫：需完成 LLM 配置才能进入
if (to.name === 'Competition') {
  const hasLLMConfig = localStorage.getItem('llm_key_mode') ||
    await checkUserHasConfig()
  if (!hasLLMConfig) {
    next({ name: 'LlmConfig', query: { redirect: to.fullPath,
      tip: '竞赛模式需要先配置LLM API密钥' } })

}
```

## 四、页面间数据传递

| 场景 | 传递方式 | 示例 |
|------|---------|------|
| 知识库→详情页 | URL 参数 `:id` | `/knowledge/15` → 加载模型ID=15详情 |
| Agent列表→对话 | URL 参数 `:type` | `/agent-chat/modeling-guide` → 激活建模引导 |
| 竞赛列表→阶段详情 | URL 参数 `:id` | `/competition/proj_abc` → 加载该项目的S0-S8 |
| 对话→工作台 | Pinia Store + SessionStorage | 对话中点击"在编辑器中打开" |
| 任何页面→登录 | query `?redirect=` | 登录后跳回原始页面 |
| Deep link | query `?stage=S5` | `/competition/proj_abc?stage=S5` 直接定位到S5 |

## 五、路由懒加载与代码分割

```
首页加载（必须）:
  - Login.vue / Register.vue  ← 小体积，直接include
  - Sidebar.vue               ← 全局布局

按需加载（路由级懒加载）:
  - Dashboard.vue
  - KnowledgeBase.vue
  - KnowledgeDetail.vue
  - Workspace.vue             ← 含 Monaco Editor (~2MB)，务必懒加载
  - AgentChat.vue             ← 含 Markdown 渲染器
  - Competition.vue
  - CompetitionStage.vue
  - Profile.vue
  - TeacherStats.vue

独立包（手动分割）:
  - react-widgets.js          ← React组件独立打包
  - monaco-worker.js          ← Monaco Editor Web Worker
```
