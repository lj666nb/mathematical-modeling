## 二、用户系统（USR）

### USR-001：用户注册

| 字段 | 内容 |
|------|------|
| **优先级** | P0 |
| **归属层** | 教学层 |
| **功能描述** | 通过邮箱/用户名+密码注册平台账号 |
| **用户故事** | 作为新用户，我想注册一个账号，以便使用平台的全部功能 |
| **前置条件** | 无 |
| **基本流程** | ① 访问 `/register` → ② 填写用户名(3-50字符)、密码(≥6字符)、角色(学生/教师)、班级(可选)、昵称(可选) → ③ 点击注册 → ④ 后端校验用户名唯一性 → ⑤ 创建用户，返回成功 → ⑥ 自动跳转登录页 |
| **异常流程** | 用户名已存在 → 提示"用户名已被注册"；密码过短 → 前端即时校验提示；网络错误 → 提示"注册失败，请稍后重试" |
| **验收标准** | □ 用户名唯一性校验生效 □ 密码 bcrypt 哈希存储(非明文) □ 注册成功后可登录 □ Playwright: 注册→登录流程通过 |
| **关联功能** | USR-002 (登录) |
| **复用情况** | ✅ 复用现有 `auth/register` 接口和 `Register.vue` 页面 |

### USR-002：用户登录 + JWT鉴权

| 字段 | 内容 |
|------|------|
| **优先级** | P0 |
| **归属层** | 教学层 |
| **功能描述** | 用户名+密码登录，返回JWT令牌用于后续请求鉴权 |
| **用户故事** | 作为已注册用户，我想登录平台，以便访问我的学习数据和配置 |
| **前置条件** | 已完成注册 USR-001 |
| **基本流程** | ① 访问 `/login` → ② 输入用户名+密码 → ③ 点击登录 → ④ 后端校验 → ⑤ 返回JWT token + 用户信息 → ⑥ 前端存入 localStorage，跳转 Dashboard |
| **异常流程** | 用户名或密码错误 → "用户名或密码错误"；Token过期 → 自动跳转登录页（401拦截器）；记住回跳路径 → 登录后跳回原始页面 |
| **验收标准** | □ JWT有效期24h □ 401自动跳转登录 □ 登录后跳回原始页面(redirect参数) □ Token持久化到localStorage |
| **关联功能** | USR-001 (注册), USR-004 (LLM配置) |
| **复用情况** | ✅ 复用现有 `auth/login` 接口和 `Login.vue` 页面 |

### USR-004：LLM API 密钥配置

| 字段 | 内容 |
|------|------|
| **优先级** | P0 |
| **归属层** | 教学层 |
| **功能描述** | 用户自行配置大模型API密钥，支持多厂商(OpenAI/DeepSeek/通义千问/GLM)、多配置 |
| **用户故事** | 作为用户，我想使用自己的大模型API密钥，以便不受平台限制自由选择模型 |
| **前置条件** | 已登录 USR-002 |
| **基本流程** | ① 访问 `/llm-config` → ② 点击"添加配置" → ③ 选择厂商(provider) → ④ 填写API密钥、Base URL、模型名称 → ⑤ 设置温度和最大Token → ⑥ 保存 → ⑦ 可设为激活配置(is_active=1) |
| **异常流程** | 密钥格式错误 → 前端校验；保存失败 → 提示错误信息 |
| **验收标准** | □ 支持4个厂商的预设Base URL □ API密钥脱敏显示(前端只显示后4位) □ 多配置可切换激活 □ 浏览器密钥模式可用 |
| **关联功能** | USR-005 (连通性测试), AGT-001~004 (Agent对话), CMP-001~014 (竞赛模式) |
| **复用情况** | ✅ 复用现有 `llm_config` 模块和 `LlmConfig.vue` 页面 |

## 九、非功能需求

### 9.1 性能要求

| 指标 | 目标值 | 测量方式 |
|------|:---:|------|
| 页面首屏加载(LCP) | < 2.5s | Lighthouse |
| 页面交互响应(FID) | < 100ms | Lighthouse |
| API 响应时间(P95) | < 500ms | 后端日志 |
| LLM 对话首token时间 | < 3s | Agent日志 |
| 代码沙箱冷启动 | < 5s | sandbox service日志 |
| 知识库搜索响应 | < 500ms | API响应时间 |
| 竞赛阶段(S1)执行 | < 120s(含LLM调用) | stage tracker |
| WebSocket 流式延迟 | < 200ms/token | 前端监控 |

### 9.2 安全要求

| 要求 | 实现方式 |
|------|---------|
| 密码安全 | bcrypt哈希存储，不记录明文 |
| JWT令牌 | 24h过期，密钥从环境变量读取 |
| LLM API密钥 | 数据库存储(后端脱敏显示)，浏览器密钥模式(密钥不出浏览器) |
| SQL注入防护 | SQLAlchemy ORM参数化查询 |
| XSS防护 | Vue3模板自动转义 + CSP头 |
| CSRF防护 | SameSite Cookie + JWT Bearer Token(非Cookie) |
| 代码沙箱隔离 | Docker容器：无网络、CPU 1核、内存512MB、磁盘1GB、超时30s强制kill |
| 文件上传安全 | 类型白名单、大小限制、病毒扫描(可选) |
| API限流 | 登录接口限流 10次/分钟/IP |

### 9.3 可用性要求

| 要求 | 实现方式 |
|------|---------|
| 响应式设计 | Element Plus 响应式栅格，适配 1366px-1920px |
| 错误提示 | 所有用户操作失败时给出可理解的中文提示(非技术栈信息) |
| 加载状态 | 所有异步操作显示Loading(骨架屏/进度条/旋转图标) |
| 空状态 | 列表无数据时显示引导性空状态("还没有项目，点击创建第一个竞赛项目") |
| 浏览器兼容 | Chrome 90+, Edge 90+, Firefox 90+ |
| 国际化 | 首期仅中文，代码和API保留英文 |

### 9.4 可维护性要求

| 要求 | 实现方式 |
|------|---------|
| 代码规范 | Python: Black + Ruff; JavaScript: ESLint + Prettier |
| 后端测试覆盖率 | 核心Service >80%, Router >60% |
| E2E测试 | Playwright 覆盖5大核心流程 |
| API文档 | FastAPI自动生成 `/docs` (Swagger UI) |
| 日志 | 结构化日志(JSON格式)，区分INFO/WARNING/ERROR |
| 配置管理 | 环境变量 `.env`，不硬编码 |
